# Anti-Patterns

<!-- Things that went wrong — avoid these -->

| Anti-Pattern                                                                                                 | Why It Fails                                                                                                                             | Do This Instead                                                                                                              |
| ------------------------------------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------- |
| Return `{}` for "no data"                                                                                    | Truthy in JS, breaks UI conditional checks                                                                                               | Return `null`                                                                                                                |
| Manual type interfaces                                                                                       | Go stale when API shapes change                                                                                                          | Use derived types from API framework                                                                                         |
| Copy-paste utility functions                                                                                 | Diverge over time, reviewer flags every time                                                                                             | Extract to shared lib on 2nd occurrence                                                                                      |
| Bare external service calls                                                                                  | Crash when service unavailable in dev                                                                                                    | try-catch + dev-mode fallback                                                                                                |
| INSERT without enum verification                                                                             | Silent failure, no error, no data                                                                                                        | Check AGENTS.md Database Enums section                                                                                       |
| Assume unique constraints exist                                                                              | Duplicates cause logic bugs                                                                                                              | Query schema to verify before seeding                                                                                        |
| Skip seed data phase                                                                                         | Pages render empty, waste hours debugging                                                                                                | Run `./loop.sh seed` before UI tasks                                                                                         |
| Defer reviewer findings indefinitely                                                                         | Same YELLOW flagged 6 times                                                                                                              | KNOWN_ISSUES.md with auto-escalate at 3                                                                                      |
| Test only with mocks, never with real data                                                                   | Misses FK chain gaps, enum mismatches                                                                                                    | Add data chain smoke tests                                                                                                   |
| `git add -A` in monorepos/submodules                                                                         | Commits deleted files, secrets, build artifacts                                                                                          | Always `git add` specific files                                                                                              |
| `.update().set()` without `.where()`                                                                         | Updates ALL rows in the table                                                                                                            | Source verification regex to ensure every `.set()` chains `.where()`                                                         |
| `.rejects.toThrow()` without error code                                                                      | Catches wrong errors, weak assertion                                                                                                     | `.rejects.toMatchObject({ code: 'FORBIDDEN' })`                                                                              |
| Register existing procedures without review                                                                  | Pre-existing bugs (missing `.where()`, no auth) go live                                                                                  | Review implementation before registration                                                                                    |
| `handleSubmit` with `getValues()`                                                                            | Uses stale form data from render time                                                                                                    | Use the validated `values` parameter from form library                                                                       |
| `useState(prop)` in mounted component                                                                        | Never updates when prop changes from async load                                                                                          | `useEffect` to sync state from prop                                                                                          |
| `.set({})` when all fields optional                                                                          | Unnecessary SQL UPDATE, wasted round-trip                                                                                                | Guard with `Object.keys(input).length === 0` early return                                                                    |
| Ignore `deletedAt` in queries                                                                                | Ghost records appear in aggregations and lists                                                                                           | ALWAYS add `isNull(deletedAt)` on soft-delete tables                                                                         |
| Read-check-write award/grant outside a transaction                                                           | Concurrent requests double-pay (XP/gems/freezes)                                                                                         | Wrap CAS + award helpers in ONE `$transaction` with a `@@unique([userId,reason,refId])` idempotence gate [from: itqan]       |
| Lock-free read-modify-write on a counter (`freezes+=n`, `balance-=n`)                                        | Lost updates under concurrency                                                                                                           | Atomic `UPDATE SET col = LEAST(col+n, MAX)`, or take `pg_advisory_xact_lock` then re-derive balance in-tx [from: itqan]      |
| `useMutation` with `onSuccess` only (no `onError`)                                                           | Silent failure, user retries blindly (RED-escalated at 3rd instance)                                                                     | Every mutation needs an error affordance (toast/inline) [from: itqan]                                                        |
| Optimistic save with no `onError` + no retry queue                                                           | Lost writes on transient failure with no user signal                                                                                     | Add `onError` + queue/retry, or block navigation until the write commits [from: itqan]                                       |
| Shipping a calc engine with zero callers (inert)                                                             | Feature silently does nothing yet passes unit tests                                                                                      | Wire it to a real path AND add an inertness regression test asserting >=1 caller [from: itqan]                               |
| Enforcing a server invariant only in the presentation layer                                                  | Bypassable via direct API call                                                                                                           | Enforce in the procedure; client checks are UX sugar only [from: itqan]                                                      |
| Dead denormalized column (seeded to a default, never read/written) + stale doc comment that contradicts impl | Future code trusts it and computes wrong values                                                                                          | Drop or wire the column; treat contradicting comments as escalate-now [from: itqan]                                          |
| Copying a multi-call-site query assembly instead of extracting                                               | Call sites diverge, cross-cutting fixes can't apply atomically                                                                           | Extract a shared helper on the 2nd occurrence (generalizes copy-paste to query assemblies) [from: itqan]                     |
| Wiring a previously-inert O(n^k) engine without bounding its input                                           | An unbounded `findMany` feeding an O(n²) dedup/similarity scan is a CPU-spike DoS on the first real dataset                              | Add a `.take(N)` cap on the upstream query in the SAME commit that wires the engine [from: itqan]                            |
| Tight `maxBuffer` on a child-process (`execFile`/`spawn`) call                                               | A verbose error/warning stream from corrupt input overflows it and throws `RangeError`, bypassing your typed `ENOENT`/UNAVAILABLE branch | Size `maxBuffer` for worst-case stderr (or document why it's enough) and keep the typed-error branch reachable [from: itqan] |
| Custom `role="button"`/`tabIndex={0}` div with `onClick` but no `onKeyDown` or `aria-label`                  | Keyboard-only and screen-reader users can focus it but Enter/Space do nothing and it announces as unlabeled                              | Add Enter/Space `onKeyDown` activation + an accessible name, or just use a real `<button>` [from: itqan]                     |
| Validating an untrusted JSON/serialized column only on write                                                 | A row poisoned out-of-band, by a migration, or by schema drift ships arbitrary JSON to clients on read                                   | `safeParse` per row on the READ path too; degrade or throw a typed error on parse failure [from: itqan]                      |
| Feeding a derived metric (crownLevel, score, rank) empty/placeholder inputs just to make the column exist    | The function returns a constant (e.g. always crown 1) and a test written to match it cements the wrong value                             | Compute the metric from real data or drop the column; never seed a derived field with degenerate inputs [from: itqan]        |
| Source test using OR-logic across independent signals (`hasLink \|\| hasRouter \|\| hasParams`)              | Passes if ANY branch is true, hiding removal of the actually-required pattern                                                            | Assert the specific required combination/attribute directly [from: ITQAN]                                                    |
| Icon-only menu/dialog/drawer trigger with no `aria-label` or visible text                                    | Unreachable to assistive tech and forces brittle test selectors                                                                          | Give the trigger an explicit accessible name [from: ITQAN]                                                                   |
