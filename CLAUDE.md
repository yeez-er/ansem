## Core Rules

### TDD Is Non-Negotiable

```
1. Write tests FIRST (Red)
2. Write minimum code to pass (Green)
3. Refactor while keeping green
4. Run FULL suite before commit
5. No commit without green
```

### What "Done" Means

- All unit tests exist and pass
- All integration tests exist and pass
- No skipped tests (`.skip`, `xit`, `@skip`)
- No TODO comments in code
- No placeholder implementations
- No `console.log` debugging left behind
- Coverage meets threshold (>= 80%)
- Build succeeds
- Lint passes

### Commit Discipline

- `git add` specific files only — NEVER `git add -A` or `git add .`
- Descriptive commit messages: `feat:`, `fix:`, `refactor:`, `test:`, `docs:`
- One task per commit. One commit per iteration.

## API Contracts

- Procedures returning "no data" MUST return `null`, not `{}` or `[]`
- `{}` is truthy in JavaScript — `!{}` is `false` — this causes real bugs
- UI consumers MUST check with proper null checks, not just truthiness

---

## External Services

- Every external service call MUST be wrapped in try-catch
- Dev mode MUST have a fallback (mock, local, or placeholder)
- No feature should crash because a third-party is unreachable
- Document all services in `ralph/AGENTS.md` External Services table

---

## Code Quality

- No duplication: if same logic exists 3+ places, extract to shared lib
- Prefer derived types (`RouterOutputs['router']['procedure']`) over manual interfaces
- Database enums: always verify valid values before INSERT (see `ralph/AGENTS.md`)
- Unique constraints: verify they exist for "one per X" relationships
